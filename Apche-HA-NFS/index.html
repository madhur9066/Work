Web Server Load Balancer with Shared NFS Storage

This demonstration shows multiple web servers serving the same website content from a shared NFS Storage. Incoming client requests are distributed across the web servers by a Global Load Balancer (HAProxy). Website content can be uploaded through an SFTP Server and is immediately available on all web servers.

Components Used

Component Purpose
Apache HTTP Server (3 Instances) Hosts and serves the website content.
Global Load Balancer (HAProxy) Distributes incoming HTTP requests across all Apache web servers.
NFS Server Provides shared storage accessed by all web servers.
SFTP Server Allows users to upload and manage website files stored on the shared NFS location.
Shared NFS Storage Ensures all web servers serve the same website content.

Environment Details

Request Flow

Step Description
1 User accesses the application through the Global Load Balancer.
2 The Load Balancer forwards the request to one of the Apache web servers.
3 The selected Apache server reads website files from the shared NFS storage.
4 Content uploaded through the SFTP server is immediately available to all web servers.
